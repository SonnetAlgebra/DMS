# 数据监控系统需求文档

## 一、项目背景与目标

### 核心定位

打造一个**个人可运行的最小化监控系统原型**，通过构建完整但轻量的系统，深入理解监控系统端到端链路的核心架构与实现逻辑。

> **执行状态**：2026-04-07，已完成设想阶段，进入执行阶段开发

### 学习方式

**边做边学，迭代验证** - 每次迭代都有可验证的输出，在实现中理解工程问题，避免纯理论学习的抽象性。

### 目标用户

**技术人员学习者** - 面向想要深入理解监控系统架构的技术人员，而非最终业务用户。系统设计以学习价值为导向，而非生产环境的复杂需求。

## 二、核心功能需求

### 数据接入（P0）

- **CSV文件导入**：支持用户上传CSV格式文件，模拟真实数据源（如Kafka、API等）
- **格式要求**：CSV需包含时间戳字段和至少一个数值型指标字段
- **数据校验**：自动识别表头，校验时间格式和数值合法性

### 数据存储（P0）

- **轻量级持久化**：使用本地数据库存储，无需额外安装服务
- **数据结构**：存储指标名称、时间戳、数值、异常标记等字段
- **查询能力**：支持按时间范围和指标类型检索历史数据

### 异常检测（P0）

- **Z-Score方法**：基于3σ准则识别离群点
- **计算公式**：$Z = \frac{x - \mu}{\sigma}$，其中μ为均值，σ为标准差
- **阈值配置**：默认阈值为3，支持用户动态调整
- **标记机制**：超过阈值的数据点标记为异常

### 多维监控（P0）

- **多指标支持**：同时监控多个业务指标（如日活、GMV、转化率等）
- **独立处理**：每个指标独立进行异常检测和存储
- **灵活扩展**：支持新增指标类型，无需修改核心逻辑

### 可视化展示（P0）

- **时序趋势图**：以时间为横轴，指标值为纵轴绘制折线图
- **异常标记**：异常数据点用红色高亮显示
- **热力图**：展示指标在不同时间段的分布密度
- **交互能力**：支持图表缩放、指标切换等基础交互

### 报警机制（P1）

- **简单规则引擎**：基于异常检测结果触发报警
- **输出方式**：控制台日志输出和本地文件记录
- **报警内容**：包含时间、指标名称、异常值等信息

### 关联分析（P1）

- **相关性计算**：支持计算两个指标间的皮尔逊相关系数
- **可视化辅助**：通过散点图展示指标间关系
- **根因定位**：帮助理解指标间的联动关系

### 扩展性设计（P1）

- **接口预留**：架构设计预留数据源替换能力
- **模块化**：各功能模块解耦，便于单独替换或升级

## 三、非功能性需求

### 运行环境

- **平台支持**：Windows、macOS、Linux均可运行
- **硬件要求**：普通个人电脑配置即可，无特殊要求
- **部署方式**：本地启动，无需容器化或复杂部署流程

### 技术架构

- **后端框架**：轻量级、支持异步处理的Web框架
- **前端框架**：响应式前端框架，支持动态数据渲染
- **数据库**：单文件本地数据库，无需额外服务
- **可视化库**：成熟的开源图表库

### 易用性

- **界面简洁**：以数据图表为核心，去除复杂菜单
- **操作流畅**：数据导入和图表刷新响应迅速
- **学习友好**：代码结构清晰，便于理解和修改

## 四、明确排除项

| 排除项 | 原因 |
| ------ |------ |

| 多用户并发 | 个人项目，单机场景足够 |
| 实时流处理 | MVP阶段用批量导入模拟 |
| 复杂报警规则 | 简化为核心阈值告警 |
| 用户认证系统 | 无需权限管理和多租户支持 |
| 分布式架构 | 单机运行即可满足学习需求 |
| 外部通知渠道 | 暂不集成邮件、短信等通知方式 |

## 五、关键术语说明

- **Z-Score（标准分数）**：衡量数据点距离平均值的标准差数，用于识别离群点
- **3σ准则**：统计学中，约99.7%的数据落在均值±3个标准差范围内
- **时序数据**：按时间顺序排列的数据点序列
- **皮尔逊相关系数**：衡量两个变量间线性相关程度的统计指标（-1到1之间）
- **CSV（逗号分隔值）**：一种简单的表格数据存储格式
