## ⚠️ 技术债务与风险

### 当前技术债务

#### 1. SQLAlchemy 兼容性问题（已记录）
- **债务描述**：计划书要求使用 SQLAlchemy 2.0+，但实际使用原生 SQLite
- **原因**：Python 3.13 与 SQLAlchemy 2.x 存在兼容性问题
- **错误信息**：`AssertionError: Class <class 'sqlalchemy.sql.elements.SQLCoreOperations'> 直接继承 TypingOnly 但有额外属性`
- **解决方案**：
  - **短期**：维持现状，使用原生 SQLite
  - **中期**：评估降级至 Python 3.12 或使用 SQLAlchemy 1.4
  - **长期**：根据项目需求决定是否恢复 ORM
- **影响评估**：开发效率降低，但功能无影响

#### 2. 已解决的约束问题（已完成）
- **问题**：原 time_series 表有 UNIQUE(metric_id, timestamp) 约束
- **解决**：移除唯一约束，允许业务层处理重复数据
- **符合性**：现在完全符合计划书 v1.5 2.4 节定义
