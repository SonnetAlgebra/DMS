# DMS 项目说明

## 📊 项目概述

**项目名称**：DMS (Data Monitoring System - 数据监控系统)

**项目目标**：构建一个时序数据监控与异常检测平台，支持 CSV 数据导入、动态异常判定（Z-Score 算法）、异常快照保存、Pearson 关联分析及报警引擎。

**业务场景**：用于各类指标数据的实时监控，自动识别异常波动，提供历史异常快照查询，支持多指标关联分析，及时触发报警通知。

### 数据模型（4 张核心表）

| 表名 | 用途 | 关键字段 |
|------|------|----------|
| `metrics` | 指标定义 | id, name, description, unit |
| `time_series` | 时序原始数据 | metric_id, timestamp, **value (允许 NULL)** |
| `anomalies` | 异常快照 | 记录检测时的阈值上下文 |
| `alerts` | 报警记录 | anomaly_id, status |

### 核心设计原则

1. **原始数据纯净性**：`time_series` 仅存储原始数据，不存储 `is_anomaly` 或 `z_score`，查询时动态计算
2. **快照快进快出**：`anomalies` 是快照表，每次 `/detect` 调用时删除该指标所有旧快照
3. **除零保护强制**：Z-Score 计算 `std=0` 时必须返回 `NULL`，不可抛异常
4. **缺失值不填充**：数据库中缺失值必须返回 `NULL`，不私自填充默认值
5. **字段完全一致**：数据模型必须与计划书字段完全一致，不私自增减

### 核心业务约束

| 约束项 | 要求 | 违规后果 |
|--------|------|----------|
| **架构锁定** | FastAPI + SQLite，禁止更换技术栈 | 立即停止并上报 |
| **除零保护** | Z-Score `std=0` 时返回 `NULL` | 立即停止并上报 |
| **缺失值处理** | 缺失值返回 `NULL`，不填充默认值 | 立即停止并上报 |
| **字段一致性** | 数据模型与计划书字段完全一致 | 立即停止并上报 |
