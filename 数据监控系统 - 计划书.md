### 2.4 数据模型设计

```sql
-- 指标定义表
CREATE TABLE metrics (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(50) NOT NULL UNIQUE,
    description VARCHAR(200),
    unit VARCHAR(20),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 时序数据表：仅存储原始数据，不存储判定结果
CREATE TABLE time_series (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    metric_id INTEGER NOT NULL,
    timestamp DATETIME NOT NULL,
    value REAL,                          -- 原始值（NULL表示缺失）
    FOREIGN KEY (metric_id) REFERENCES metrics(id),
    UNIQUE(metric_id, timestamp)
);

-- 异常记录表：报警快照，记录触发时的上下文
-- 策略：每次/detect调用时，删除该指标所有旧快照，再保存当前阈值下的异常点
-- 语义：快照始终代表"最后一次检测时的判定结果"
CREATE TABLE anomalies (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    metric_id INTEGER NOT NULL,
    timestamp DATETIME NOT NULL,
    value REAL NOT NULL,
    z_score REAL NOT NULL,               -- 当时的Z-Score得分
    threshold_used REAL NOT NULL,        -- 当时使用的阈值
    mean_used REAL NOT NULL,             -- 当时使用的均值
    std_used REAL NOT NULL,              -- 当时使用的标准差
    detected_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (metric_id) REFERENCES metrics(id)
);

-- 报警记录表
CREATE TABLE alerts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    anomaly_id INTEGER NOT NULL,
    metric_id INTEGER NOT NULL,
    message TEXT,
    status VARCHAR(20) DEFAULT 'pending',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (metric_id) REFERENCES metrics(id)
);

-- 索引优化
CREATE INDEX idx_time_series_metric_time ON time_series(metric_id, timestamp);
CREATE INDEX idx_anomalies_metric ON anomalies(metric_id);
CREATE INDEX idx_alerts_status ON alerts(status);
```

### 2.5 动态异常判定逻辑

```python
# 查询时根据当前阈值动态计算
def detect_anomalies(metric_id, start, end, threshold=3.0):
    # 1. 查询原始数据（带分页限制）
    data = query_time_series(metric_id, start, end, limit=5000)
    
    # 2. 边界判断：有效数据点 < 2
    values = [d.value for d in data if d.value is not None]
    if len(values) < 2:
        return {"error": {"code": "INSUFFICIENT_DATA", "message": "有效数据点不足2个"}}
    
    # 3. 计算统计量
    mean = np.mean(values)
    std = np.std(values)
    
    # 4. 边界判断：标准差为0
    if std == 0:
        z_scores = [0] * len(values)
    else:
        z_scores = [abs((v - mean) / std) for v in values]
    
    # 5. 动态判定（不存储结果）
    anomalies = []
    result_data = []
    
    for i, d in enumerate(data):
        if d.value is None:
            result_data.append({
                "timestamp": d.timestamp,
                "value": None,
                "is_anomaly": False,
                "z_score": 0
            })
            continue
        
        is_anomaly = z_scores[i] > threshold if std > 0 else False
        
        result_data.append({
            "timestamp": d.timestamp,
            "value": d.value,
            "is_anomaly": is_anomaly,
            "z_score": z_scores[i]
        })
        
        if is_anomaly:
            anomalies.append({
                "timestamp": d.timestamp,
                "value": d.value,
                "z_score": z_scores[i],
                "threshold": threshold
            })
    
    return {
        "data": result_data,
        "anomalies": anomalies,
        "mean": mean,
        "std": std
    }

# 异常快照保存策略：先删除后插入
def save_anomaly_snapshots(metric_id, anomalies, threshold, mean, std):
    """
    保存异常快照策略：
    1. 删除该指标下所有旧快照
    2. 批量插入新快照
    语义：快照始终代表"最后一次检测时的判定结果"
    """
    # 1. 删除旧快照
    db.execute("DELETE FROM anomalies WHERE metric_id = ?", (metric_id,))
    
    # 2. 批量插入新快照
    for anomaly in anomalies:
        db.execute("""
            INSERT INTO anomalies 
            (metric_id, timestamp, value, z_score, threshold_used, mean_used, std_used)
            VALUES (?, ?, ?, ?, ?, ?, ?)
        """, (metric_id, anomaly["timestamp"], anomaly["value"], 
              anomaly["z_score"], threshold, mean, std))
    
    db.commit()
```

### 2.6 API接口设计

| 方法 | 路径 | 描述 | 请求/响应 |
|------|------|------|-----------|
| POST | `/api/v1/data/upload` | 上传CSV文件，导入数据 | `multipart/form-data` → `{success, inserted, metrics}` |
| GET | `/api/v1/metrics` | 获取所有指标列表 | → `{metrics: [{id, name, description}]}` |
| GET | `/api/v1/metrics/{id}/data` | 获取指定指标的历史数据（含动态异常判定） | `?start=&end=&threshold=3.0&limit=1000&offset=0` → `{data: [{timestamp, value, is_anomaly, z_score}], mean, std}` |
| POST | `/api/v1/detect` | 触发异常检测并保存快照 | `{metric_ids, threshold}` → `{anomalies, count}` |
| GET | `/api/v1/anomalies` | 获取异常记录（历史快照） | `?metric_id=` → `{anomalies, total}` |
| POST | `/api/v1/correlation` | 执行指标关联分析 | `{metric_ids, method, resample="D"}` → `{matrix, metrics}` 或 `{error: {code, message}}` |

**关联分析重采样说明**：
- `resample` 参数指定重采样频率，默认 `"D"`（按天）
- 支持值：`"H"`（小时）、`"D"`（天）、`"W"`（周）
- 对齐规则：各指标按指定频率**取均值**后，取时间交集计算相关性
- 有效时间点少于3对时返回错误：`{error: {code: "INSUFFICIENT_DATA", message: "有效数据点不足3对"}}`

## 三、功能需求

### 3.1 P0 核心功能（MVP必选）

| 功能模块 | 需求描述 | 验收标准 |
|----------|----------|----------|
| **数据接入** | 支持CSV文件上传导入 | 能正确解析CSV，自动识别时间戳和数值字段 |
| **数据存储** | SQLite轻量级持久化，仅存储原始数据 | 数据成功存入，NULL值保留不填充 |
| **异常检测** | Z-Score方法（3σ准则），查询时动态计算 | 阈值可配置，默认3.0，判定结果不存储 |
| **多维监控** | 支持多指标独立处理 | 可同时监控日活、GMV、转化率等指标 |
| **可视化展示** | 时序图 + 异常标记 + 断点展示 | ECharts渲染，异常点红色高亮，缺失值显示为断点 |

> **Z-Score（标准分数）**：衡量数据点距离平均值的标准差数，计算公式为 `Z = (x - μ) / σ`。当\|Z\| > 阈值时判定为异常。
>
> **3σ准则**：统计学原理，约99.7%的正态分布数据落在均值±3个标准差范围内，是Z-Score判定异常的依据。
>
> **数据断点**：时序数据中出现NULL值的区间，前端图表不连线，直观展示数据缺失。

### 3.2 P1 扩展功能（MVP后迭代）

| 功能模块 | 需求描述 | 验收标准 |

| **关联分析** | Pearson相关系数，热力图展示 | 支持多指标相关性分析，有效点对<3时返回错误 |

> **Pearson相关系数**：衡量两个变量间线性相关程度的统计指标，取值范围-1到1。

```
    ├─► 后端: 查询原始数据（limit=5000）→ 动态计算Z-Score → 返回带is_anomaly标记的数据
    │
    ├─► 返回: {data: [{timestamp, value, is_anomaly, z_score}], mean, std}
    │
    ▼
前端渲染时序图
    │
    ├─► 正常数据：连线
    ├─► 异常数据：红点标记
    └─► 缺失数据：断点（connectNulls: false）
    │
    ▼
用户点击"检测异常"
    │
    ├─► 前端: 调用 /api/v1/detect
    │
    ├─► 后端: 执行Z-Score计算 → 删除旧异常记录 → 保存新异常快照到anomalies表
    │
    ├─► 返回: 异常点列表
    │
    ▼
异常面板显示异常详情
```

## 附录B：一键启动脚本

### start.sh（Mac/Linux）

```bash
#!/bin/bash

# 颜色定义
RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
NC='\033[0m' # No Color

echo "🔍 检查环境..."

# 检查 Python
if ! command -v python3 &> /dev/null; then
    echo -e "${RED}❌ 未检测到 Python3，请先安装 Python 3.10+${NC}"
    echo "   安装指南: https://www.python.org/downloads/"
    exit 1
fi
PYTHON_VERSION=$(python3 --version 2>&1 | awk '{print $2}')
echo -e "${GREEN}✅ Python3: $PYTHON_VERSION${NC}"

# 检查 Bun
if ! command -v bun &> /dev/null; then
    echo -e "${RED}❌ 未检测到 Bun，请先安装 Bun${NC}"
    echo "   安装命令: curl -fsSL https://bun.sh/install | bash"
    exit 1
fi
BUN_VERSION=$(bun --version 2>&1)
echo -e "${GREEN}✅ Bun: $BUN_VERSION${NC}"

# 检查虚拟环境
if [ ! -d "backend/venv" ]; then
    echo -e "${YELLOW}⚠️  未检测到虚拟环境，正在创建...${NC}"
    cd backend
    python3 -m venv venv
    cd ..
fi

# 启动后端
echo "🚀 启动后端服务 (http://127.0.0.1:8000)..."
cd backend
source venv/bin/activate
pip install -q -r requirements.txt
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000 &
BACKEND_PID=$!
cd ..

# 启动前端
echo "🚀 启动前端服务 (http://localhost:5173)..."
cd frontend
bun install &> /dev/null
bun run dev &
FRONTEND_PID=$!
cd ..

echo ""
echo -e "${GREEN}✅ 服务已启动${NC}"
echo "   📡 后端API: http://127.0.0.1:8000"
echo "   📡 前端页面: http://localhost:5173"
echo "   📚 API文档: http://127.0.0.1:8000/docs"
echo ""
echo "按 Ctrl+C 停止所有服务"

# 捕获中断信号
trap "echo ''; echo '🛑 正在停止服务...'; kill $BACKEND_PID $FRONTEND_PID 2>/dev/null; exit" INT
wait
```
